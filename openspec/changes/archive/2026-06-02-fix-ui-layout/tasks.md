# UI布局修复任务列表

## 任务概述

修复发票批量重命名工具的界面显示问题，包括空白行和布局调整功能缺失。

## 任务列表

### 任务1：移除固定高度设置
**优先级**：高  
**预估时间**：30分钟  
**状态**：已完成

**描述**：移除所有Treeview和Listbox组件的固定height设置，让组件完全由容器控制大小。

**具体步骤**：
1. 修改 `src/invoice_renamer/gui/format_builder.py`：
   - 移除第63行 `height=8` 设置
   - 移除第95行 `height=6` 设置
2. 修改 `src/invoice_renamer/gui/excel_viewer.py`：
   - 移除第22行 `height=15` 设置
3. 确保所有组件使用 `fill=tk.BOTH, expand=True` 打包

**验收标准**：
- 所有组件不再有固定高度设置
- 组件能够自适应容器大小
- 内容不足时不再显示空白行

---

### 任务2：添加垂直PanedWindow
**优先级**：高  
**预估时间**：45分钟  
**状态**：已完成

**描述**：在左侧区域内部添加垂直方向的PanedWindow，实现数据预览区域和格式构建器之间的垂直调整功能。

**具体步骤**：
1. 修改 `src/invoice_renamer/gui/main_window.py`：
   - 在 `_create_widgets` 方法中，在 `left_frame` 内添加垂直PanedWindow
   - 将 `data_frame` 和 `format_frame` 添加到垂直PanedWindow
   - 配置权重比例：数据预览区域 weight=7，格式构建器 weight=3
   - 设置格式构建器最小高度：minsize=80
2. 调整现有的布局代码，确保兼容性

**验收标准**：
- 左侧区域内部出现垂直分隔条
- 用户可以上下拖动分隔条调整大小
- 初始比例为70:30
- 格式构建器最小高度为80像素

---

## 风险评估

### 技术风险
1. **低风险**：移除固定高度设置可能影响某些组件的初始显示
2. **中风险**：垂直PanedWindow的集成可能需要调整现有布局代码
3. **低风险**：组件打包方式调整可能影响滚动条行为

### 缓解措施
1. 逐步实施，每完成一个任务进行测试
2. 保持现有功能的兼容性
3. 在测试阶段充分验证各种场景
