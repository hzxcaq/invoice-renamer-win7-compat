## Context

当前主窗口（`MainWindow`）使用混合布局：`main_frame`使用Grid将左右区域分列，左侧区域（`left_frame`）使用Pack垂直排列各组件。其中`data_frame`区域需要并排显示Excel预览和PDF文件列表。

现有实现中，`data_frame`使用Grid布局，两列都设置`weight=1`等分空间。但由于Excel Treeview的列宽计算逻辑（每列最小100px），当Excel文件有较多列时，Treeview请求的宽度会超出分配的空间，Grid布局会将PDF文件列表压缩到0宽度。

## Goals / Non-Goals

**Goals:**
- 解决Excel预览挤压PDF文件列表的问题
- 允许用户手动调整两个区域的宽度比例
- 保持现有功能完整性（Excel预览、PDF列表、格式构建器等）
- 提升用户体验，确保两个区域都能正常显示

**Non-Goals:**
- 不改变现有的业务逻辑（文件匹配、重命名等）
- 不改变窗口的整体结构（左右两栏布局）
- 不添加新的功能特性

## Decisions

### 1. 使用ttk.PanedWindow替代Grid布局

**决策**: 将`data_frame`中的Grid布局替换为`ttk.PanedWindow`（水平方向）

**理由**:
- `ttk.PanedWindow`专门用于创建可调整大小的窗格布局
- 用户可以通过拖动分割条自由调整两个区域的比例
- 完全避免Grid布局中的空间竞争问题
- Tkinter内置组件，无需额外依赖

**替代方案考虑**:
- **方案A（设置Grid minsize）**: 简单但不够灵活，用户无法调整比例
- **方案B（限制列宽）**: 可以缓解问题但无法根本解决空间竞争

### 2. 设置合理的默认权重和最小宽度

**决策**: 
- Excel预览窗格默认权重为3，PDF列表窗格默认权重为1
- Excel预览窗格最小宽度为400px，PDF列表窗格最小宽度为300px

**理由**:
- 默认比例3:1确保Excel预览有足够空间显示数据
- 最小宽度设置防止窗格被压缩到不可用状态
- 权重设置允许用户调整比例时，两个窗格按比例变化

### 3. 限制Excel Treeview列宽最大值

**决策**: 修改`excel_viewer.py`中的列宽计算，设置每列最大宽度为160px

**原始代码**:
```python
width = max(100, len(header) * 10)
```

**修改后**:
```python
width = min(160, max(80, len(header) * 10))
```

**理由**:
- 防止单个列过宽（如长表头名称导致列宽过大）
- 保持列宽在合理范围内，确保多列时总宽度可控
- 仍然允许内容通过横向滚动查看

### 4. 保持现有的pack和Grid混合布局

**决策**: 仅修改`data_frame`内部的布局方式，不改变`left_frame`的整体Pack布局

**理由**:
- 减少变更范围，降低引入新问题的风险
- 现有混合布局工作正常，无需完全重构
- PanedWindow可以很好地嵌入到现有的Pack布局中

## Risks / Trade-offs

### 1. 用户可能需要适应新的交互方式
**风险**: 用户习惯了固定布局，可能需要学习如何拖动分割条
**缓解**: PanedWindow的分割条是直观的交互元素，用户通常能快速适应

### 2. 初始窗口大小可能影响布局
**风险**: 不同屏幕分辨率下，初始分割比例可能不合适
**缓解**: 通过合理的默认权重和最小宽度设置，确保在大多数屏幕上都能正常显示

### 3. PanedWindow状态不持久化
**风险**: 用户调整的比例在程序重启后会重置
**缓解**: 这是可接受的限制，不影响核心功能。如需持久化，可作为后续增强

## Migration Plan

### 实施步骤
1. 修改`main_window.py`中的`data_frame`布局代码
2. 修改`excel_viewer.py`中的列宽计算逻辑
3. 测试在不同Excel列数下的布局表现
4. 验证现有功能（文件选择、Excel预览、格式构建等）不受影响

### 回滚策略
- 如果出现问题，可以快速回滚到Grid布局
- 保留原始代码注释，便于回滚

### 测试策略
- 测试不同列数的Excel文件（7列、10列、15列、20列）
- 测试窗口缩放时的布局表现
- 测试分割条拖动功能
- 验证现有功能完整性

## Open Questions

### 1. 是否需要持久化分割比例？
当前设计不持久化用户调整的分割比例。如果需要，可以将比例保存到配置文件中。

### 2. 是否需要为PDF列表添加最小宽度限制？
当前设计为PDF列表设置最小宽度300px，但可能需要根据实际使用情况调整。

### 3. 是否需要考虑高DPI屏幕？
当前设计没有特别处理高DPI屏幕，但Tkinter通常能自动处理。
