## Context

当前 Excel 文件结构：
- `发票基础信息` sheet: 132行，每行一张发票，列包含序号、发票代码、发票号码、数电发票号码、销方名称、金额、税额、价税合计等 19 列
- `信息汇总表` sheet: 242行（132个唯一发票号），多出 8 列：税收分类编码、特定业务类型、**货物或应税劳务名称**、规格型号、单位、数量、单价、税率
- `建筑服务` sheet: 3行，多出 3 列：跨区域涉税事项报验管理编号、建筑服务发生地、建筑项目名称

三个 sheet 的发票号码完全一致（建筑服务是子集），可以安全 JOIN。

商品名称格式为 `*分类名*商品名`，如 `*黑色金属冶炼压延品*镀锌管`，需要提取 `镀锌管`。

## Goals / Non-Goals

**Goals:**
- 读取所有 sheet 并按发票号码合并为一行一条发票
- 取金额最大的明细行的商品名称（去分类前缀）
- 合并建筑服务的独有列
- 格式构建器用树状结构按 sheet 分组展示列
- 单 sheet 的旧 Excel 文件仍然兼容

**Non-Goals:**
- 不支持让用户选择多条明细（只取金额最大的一条）
- 不支持实时从 PDF 内容提取信息
- 不改变现有的匹配和重命名逻辑

## Decisions

### 决策1：合并策略 — 以发票基础信息为底

**选择**: 发票基础信息作为主表（132行），LEFT JOIN 信息汇总表和建筑服务

**理由**:
- 发票基础信息每行一张发票（1:1），是天然的主键表
- 信息汇总表有重复行（1:N），需要聚合后才能 JOIN
- 建筑服务只有 3 行，是子集

**替代方案**:
- 以信息汇总表为底（需要 GROUP BY，更复杂）

### 决策2：多明细处理 — 取金额最大的一条

**选择**: 同一发票号码有多条明细时，取金额最大的那行的商品名称

**理由**:
- 用户明确要求"取金额最大的"
- 金额最大的通常是主要商品，最具代表性
- 实现简单：按发票号码分组，取 max(金额) 对应行

**替代方案**:
- 取第一条（简单但可能不准确）
- 聚合拼接所有商品名（文件名过长）

### 决策3：商品名提取 — 去掉分类前缀

**选择**: 按 `*` 分割，取最后一个非空段

**理由**:
- `*黑色金属冶炼压延品*镀锌管` → `镀锌管`
- `*酒*酒汾酒` → `酒汾酒`
- 分类名冗余，商品名才是用户关心的

**实现**: `name.split('*')[-1].strip()` 或正则 `r'\*[^*]+\*(.+)'`

### 决策4：树状选择器 — ttk.Treeview + tag

**选择**: 使用 `ttk.Treeview` 实现树状列选择，用 tag 标记选中状态

**理由**:
- Treeview 原生支持树状展开/折叠
- tag 方式实现 checkbox 效果，比嵌入 Checkbutton 更简洁
- 与现有 Tkinter 代码风格一致

**交互设计**:
```
▼ 发票基础信息 (13列)        已选列 (按顺序):
   [✓] 序号                   1. 销方名称
   [ ] 发票代码               2. 商品名称
   [✓] 数电发票号码           3. 金额
   [✓] 销方名称               4. 价税合计
   [✓] 金额
   ...                        格式预览:
▶ 信息汇总表 (1列)            {销方名称}_{商品名称}_{金额}
   [✓] 商品名称
▶ 建筑服务 (2列)
   [ ] 建筑服务发生地
   [ ] 建筑项目名称
```

- 默认展开"发票基础信息"，折叠其他分组
- 点击行即切换选中/取消选中
- 选中的行高亮显示（tag 配色）
- 双击或点击 "添加>>" 将选中列加入已选列表

### 决策5：列命名 — 使用原始列名

**选择**: 所有列保留原始列名，不创建新列名。`货物或应税劳务名称` 保持原名，值仍提取商品名（去分类前缀）

**理由**:
- 与 Excel 原始数据保持一致，用户更熟悉
- 避免混淆：列名就是 Excel 里的列名
- 信息汇总表所有 8 个独有列都合并进来：货物或应税劳务名称、规格型号、单位、数量、单价、税率、税收分类编码、特定业务类型

**交互设计**:
```
▼ 发票基础信息 (19列)       已选列 (按顺序):
   [ ] 序号                  1. 销方名称
   [ ] 发票代码              2. 货物或应税劳务名称
   [ ] 数电发票号码          3. 金额
   [ ] 销方名称              4. 价税合计
   [ ] 金额
   ...                       格式预览:
▶ 信息汇总表 (8列)           {销方名称}_{货物或应税劳务名称}_{金额}
   [ ] 货物或应税劳务名称
   [ ] 规格型号
   [ ] 单位
   [ ] 数量
   [ ] 单价
   [ ] 税率
   [ ] 税收分类编码
   [ ] 特定业务类型
▶ 建筑服务 (3列)
   [ ] 建筑服务发生地
   [ ] 建筑项目名称
```

**合并规则**:
- 重复列（如序号、发票号码、金额等）只在"发票基础信息"下显示
- 信息汇总表只显示其独有的 8 列
- 建筑服务只显示其独有的 3 列

## Risks / Trade-offs

### 风险1: 金额为 0 或负数的明细
**风险**: 某些发票的明细金额可能为 0 或负数
**缓解**: 取 max 金额时自然处理，负数不会被选中；如果全部为 0，取第一条

### 风险2: 商品名称为空
**风险**: 某些明细的"货物或应税劳务名称"可能为空
**缓解**: 跳过空值，取下一个有值的明细；全部为空则留空

### 风险3: 文件名过长
**风险**: 商品名 + 销方名 + 金额可能超过 Windows 260 字符限制
**缓解**: 复用现有的 `sanitize_filename` 逻辑，限制总长度

### 风险4: Treeview 交互复杂度
**风险**: 树状选择器比扁平 Listbox 更复杂
**缓解**: 保持已选列区域不变（仍用 Listbox），只改左侧选择区域
